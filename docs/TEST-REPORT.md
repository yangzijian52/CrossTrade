## 代码级验证

- 商品购买使用单商品锁、版本号和状态机。
- 并发更新只接受预期版本，防止最后库存重复售出。
- 扣款失败释放预留。
- 发放失败恢复背包快照并退款。
- 退款失败写入 refunds 并进入 REVIEW。
- 卖家付款使用 PENDING/PAYING/PAID/REVIEW。
- 到期写入永久邮箱；主动下架保留托管库存，删除时才写入邮箱，支持补货和重新上架。
- 面对面交易拦截 Shift、拖拽、数字键、双击、丢弃和副手交换。
- 面对面交易最终复核余额与背包空间，并按净额结算。
- 面对面交易 GUI 使用两阶段初始化，避免任一发送方向在创建第二个 GUI 前读取空引用。
- 卖家在线时即时收到成交明细；离线成交在下次上线补发通知。
- Floodgate 上架物品选择使用内置 Paper 26.2 中文名称表。
- 玩家可见商品、销售和邮箱来源状态统一转换为中文，数据库内部英文枚举保持不变。
- 面对面交易倒计时使用聊天消息，不再发送 ActionBar。
- 23 个箱子页面均有独立 `gui.yml` 节点；静态按钮支持槽位、材质、名称、Lore、权限和命令动作。
